# Steering Rules

This folder contains project rules and guidelines for the Laravel Monitoring System.

## Purpose

The steering folder stores documentation that guides development practices, coding standards, and architectural decisions for this project.

## Contents

- `coding-standards.md` - Project coding standards and naming conventions
- `component-structure.md` - Component folder structure and organization guidelines

## Usage

Developers should refer to these documents when:
- Adding new features
- Creating new components
- Making architectural decisions
- Reviewing code

All guidelines in this folder should be followed to maintain code quality and consistency.
